# 技术栈推荐 (Tech Stack Recommendation)

针对《铁窗之言》这款以**对话驱动、UI交互密集、逻辑推理为主**的2D游戏，以下是根据开发效率和功能匹配度推荐的技术栈：

## 1. 核心引擎：Godot Engine (4.x)
**理由：**
*   **出色的UI系统：** Godot 的 `Control` 节点系统是目前所有主流引擎中处理复杂 UI（如档案夹、证词本、滚动对话框）最直观、强大的。
*   **轻量级且开源：** 引擎体积小，启动快，完全免费且无分成压力。
*   **2D 性能优异：** 对于此类不依赖重度 3D 渲染的游戏，Godot 的 2D 渲染器非常高效。

## 2. 脚本语言：GDScript
**理由：**
*   语法类似 Python，学习曲线平缓。
*   与引擎深度集成，处理游戏逻辑（如证据对比、数值增减）非常快速。
*   *备选：* 如果追求极致性能或已有 C# 背景，Godot 也完美支持 **C#**。

## 3. 对话与剧情系统：Dialogue Manager (插件) 或 Ink
**理由：**
*   **Dialogue Manager：** Godot 社区非常流行的插件，支持简单的文本编写对话分支，易于集成。
*   **Ink (by inkle)：** 如果剧本极其庞大且分支极其复杂，推荐使用 Ink 编写剧情脚本，它能处理复杂的逻辑状态跳转，并有成熟的 Godot 集成方案。

## 4. 数据管理：JSON / Godot Resources
**理由：**
*   **JSON：** 用于存储静态的案件信息、证据描述和嫌疑人配置，易于阅读和编辑。
*   **Resources：** Godot 特有的资源格式，非常适合在编辑器内直接配置嫌疑人的各项属性（如初始压力值、立绘路径）。

## 5. 美术与音效工具
*   **美术：** **Adobe Photoshop** (标准) 或 **Aseprite** (如果采用像素风)。
*   **UI 设计：** **Figma** (先在 Figma 中设计 UI 布局和交互逻辑，再到引擎中还原)。
*   **音频：** **Audacity** 或 **Reaper** (用于处理环境氛围音和音效)。

## 6. 版本控制：Git + GitHub/Gitee
**理由：**
*   独立开发或小团队协作的必备工具。
*   Godot 的文件格式（.tscn, .gd）大多是基于文本的，对 Git 非常友好。

---

### 总结方案：
*   **主方案 (推荐)：** Godot + GDScript + Dialogue Manager (最快产出原型)
*   **备选方案 (职业级)：** Unity + C# + Fungus/Ink (适合已有 Unity 经验且考虑多平台发布)
