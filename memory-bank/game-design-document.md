# 游戏设计文档 (Game Design Document)

## 1. 游戏概述 (Game Overview)
**游戏名称：** (暂定) 铁窗之言 / The Interrogation Room
**游戏类型：** 逻辑推理 / 心理博弈 / 交互式小说
**核心玩法：** 通过与嫌疑人对话、出示证据、寻找证词矛盾，最终攻破嫌疑人的心理防线，还原案件真相。
**美术风格：** 2D / 黑色电影风格 (Film Noir) / 灰暗色调，以人物立绘和审讯室背景为主。

## 2. 故事背景 (Story Background)
玩家扮演一名经验丰富的刑警，面对一系列错综复杂的案件。每个案件只有一名或多名关键嫌疑人被带到审讯室。玩家需要在有限的时间和审讯次数内，通过问询和抛出线索，从谎言中榨取真相。

## 3. 核心机制 (Core Mechanics)
### 3.1 询问与分支对话 (Interrogation & Dialogue)
*   玩家可以选择不同的询问语气（如：温和、威逼、诱导）。
*   嫌疑人的回答会根据玩家的语气和当前的情绪状态（压力值/防备心）产生变化。

### 3.2 证词记录与分析 (Testimony Analysis)
*   嫌疑人的关键证词会被记录在“证词本”中。
*   玩家需要仔细阅读证词，寻找逻辑漏洞或与已知线索相悖的地方。

### 3.3 证据出示 (Presenting Evidence)
*   玩家拥有一个“档案夹”，存放案发现场收集到的证据。
*   在恰当的对话节点，出示正确的证据来反驳嫌疑人的谎言（类似《逆转裁判》的“异议！”）。
*   出示错误证据会导致嫌疑人警惕性提高，增加审讯难度。

### 3.4 心理防线/压力值系统 (Stress System)
*   嫌疑人拥有一个隐形的“压力/防线”条。
*   成功指出矛盾、出示关键证据会大幅增加压力值。
*   当压力值达到临界点，嫌疑人将“崩溃”并交代关键真相。

## 4. 游戏流程 (Game Flow)
1.  **案件简报：** 玩家阅读案情摘要，获取初始线索和证据。
2.  **开始审讯：** 嫌疑人入座，开始第一轮基础问话。
3.  **搜集证词：** 引导嫌疑人给出案发时间段的行踪和陈述。
4.  **攻防阶段：** 针对可疑证词进行深度盘问，适时抛出证据。
5.  **真相大白：** 击溃嫌疑人心理防线，听取完整真相。
6.  **结案评价：** 根据审讯效率和准确率给出案件评分。

## 5. UI与界面设计 (UI Design)
*   **主视点：** 第一人称视角，面对桌子对面的嫌疑人。
*   **左侧/下方：** 对话框和选项按钮。
*   **右侧滑出：** “档案夹”（证据）和“证词本”。
*   **隐藏UI：** 嫌疑人的微表情反馈（用于提示玩家当前的审讯策略是否有效）。

## 6. 音效与音乐 (Audio)
*   **环境音：** 时钟的滴答声、通风管的声音、审讯室的幽闭感。
*   **音乐：** 悬疑、压抑的低音BGM。在嫌疑人即将崩溃或出示关键证据时，音乐节奏加快，烘托紧张气氛。
